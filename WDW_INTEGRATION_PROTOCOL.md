# Wheeler–DeWitt (WDW) Integration Protocol

Created: 2026-02-28  
Purpose: safely integrate WDW into this study/research system without losing testability.

---

## 1) Role of WDW in this framework

WDW is treated as a **foundational diagnostic tool** for GR↔QM mismatch, not a default engine for near-term claims.

**Status:** P2 (Advanced/Secondary)  
**Default phase:** Day 27+ (orientation-level unless prerequisites are fully met)

---

## 2) Entry Prerequisites (must pass before deep use)

Proceed only if all boxes are true:

- [ ] Cluster A readiness (classical dynamics, constraints, Hamiltonian formulation basics)
- [ ] Cluster E readiness (field theory and symmetry/constraint intuition)
- [ ] QM formalism comfort (operators, Hilbert space, constraint equations)
- [ ] GR basics comfort (ADM-style decomposition at conceptual level)
- [ ] Can explain diffeomorphism invariance and why constraints appear
- [ ] Can explain (aloud, 60–90s) why “problem of time” emerges in canonical quantum gravity

If any item fails: step back to prerequisite map before WDW calculations.

---

## 3) Allowed Use-Cases

## A) Orientation Use (default)
Use WDW to:
- understand frozen formalism (`ĤΨ = 0`),
- compare external-time QM vs relational/correlational notions of time,
- map where Schrödinger evolution assumptions break.

**Output expectation:** conceptual notes + validity boundaries only.

## B) Model-Building Use (restricted)
Allowed only if tied to measurable proxy outcomes and explicit uncertainty budgets.

**Required outputs before claiming progress:**
1. EFT cutoff/validity envelope,
2. observable proxy map,
3. null-test pathway,
4. uncertainty decomposition (stat/sys/epistemic),
5. RSI score assignment.

## 5) RSI Gate for WDW-Derived Claims

Use project RSI bands:
- **RSI ≥ 80:** claims can be advanced (with replication plan)
- **60–79:** conditional; require robustness upgrades
- **< 60:** halt interpretation; recovery/replication first

WDW-heavy proposals with high epistemic uncertainty are not exempt.

---

## 6) Stop Conditions (anti-sterile-abstraction rules)

Pause WDW work immediately if any condition is met:

1. No explicit observable proxy after 2 iterations.
2. Validity range/cutoff remains undefined.
3. Claims rely on unconstrained fine-tuning.
4. Interpretation survives only under fragile priors.
5. Conceptual language expands while falsifiability decreases.

When triggered, switch to:
- prerequisite strengthening,
- proxy-first reformulation,
- or retirement to background reading.

---

## 7) Mistake Intelligence Integration

Tag WDW study mistakes with high granularity:
- `CG-WDW-Time` (problem of time confusion)
- `CG-WDW-Constraint` (Hamiltonian constraint origin confusion)
- `MF-WDW-Validity` (missed validity envelope)
- `IN-WDW-Proxy` (no measurable proxy path)
- `IN-WDW-Epistemic` (model-assumption fragility)

If `CG-WDW-*` appears 3+ times/week: recovery week required.

---

## 8) Recommended Workflow (lightweight)

1. 20–30 min orientation read (WDW source notes).  
2. 15 min “explain aloud” checkpoint (time problem + constraints).  
3. 20 min proxy-test template attempt.  
4. Log mistakes/tags.  
5. Decide: proceed / pause / roll back by RSI + stop conditions.

---

## 9) Bottom Line

WDW is essential for foundational literacy but should be treated as a **controlled high-risk module**.  
In this system, WDW is useful only when it increases testable constraint pressure rather than conceptual ornamentation.
