# GR↔QM Testability Blueprint
## An EFT-Disciplined Research Strategy for Rapid, Falsifiable Progress

Created: 2026-02-28

---

## Executive Abstract

General Relativity (GR) and Quantum Mechanics (QM) remain unmatched in their own regimes but unresolved as a unified, fully testable framework. This blueprint reframes the problem as a **feedback optimization challenge**: reduce the time and ambiguity between theoretical proposal and empirical adjudication.

Core strategy:
- prioritize **precision + modular experiments** over single mega-infrastructure dependence,
- require **EFT-first model discipline** with explicit validity ranges,
- enforce **null-test and replication standards** as primary progress engines,
- track progress via formal metrics (including a defined **Research Stability Index, RSI**).

---

## 1) Interface Regimes for Quantum-Gravity Probes

## 1.1 Precision Frontier (near-term, rapid-cycle)
**Platforms**: optical clocks, atom interferometers, EDM searches, tabletop optomechanics, short-range gravity tests.  
**Strategic value**: decouples progress from decade-scale collider timelines; probes suppressed operators through extreme precision.

## 1.2 Multi-Messenger Cosmology (high-curvature proxy regime)
**Channels**: CMB (including B-mode structures), stochastic GW backgrounds, large-scale structure, neutrino/astroparticle correlations.  
**Strategic value**: uses early-universe/high-curvature epochs as natural laboratories; cross-channel constraints reduce single-pipeline bias.

## 2) EFT-First Modeling Requirements

All model programs must satisfy the following minimum requirements before prioritization:

1. **Cutoff clarity**  
   - Explicit EFT cutoff \(\Lambda\) declaration.
   - Operator basis and coefficient priors documented.

2. **Validity envelope**  
   - Explicit regime of applicability (energy, curvature, scale, coupling assumptions).
   - Breakdown boundaries stated (where approximation fails).

3. **Observable proxy map**  
   - Each theoretical claim mapped to at least one measurable signature with expected sign/magnitude scaling.
   - Include detectability tier: current / near-term modular upgrade / long-horizon.

4. **Null-test statement**  
   - Concrete near-term exclusion pathway defined before exploratory extensions.

## 3) Falsification and Inference Framework

## 3.1 Null-Test Protocol (mandatory)
For each BSM/GR↔QM interface hypothesis:
- define null observable(s),
- define required sensitivity and confidence thresholds,
- define pre-registered acceptance/rejection logic.

## 3.2 Uncertainty Budget Standard
Every major claim must publish partitioned uncertainty:
- **Statistical**: event counts, variance model, sampling assumptions,
- **Systematic**: detector/environment/calibration effects,
- **Epistemic (model-based)**: prior dependence, model class assumptions, inverse-problem non-uniqueness.

## 3.3 Replication Rule
- independent secondary analysis team required for high-impact claims,
- blinded or semi-blinded protocol strongly preferred,
- discrepancy report required when pipelines diverge.

## 4) Formal Metrics

## 4.1 Research Stability Index (RSI)
A normalized score \([0,100]\) that measures pipeline robustness and claim reliability.

\[
\text{RSI} = 100 \times \left(0.35S_{\text{stat}} + 0.35S_{\text{sys}} + 0.20S_{\text{epi}} + 0.10S_{\text{rep}}\right)
\]

Where each component is scaled to \([0,1]\):
- \(S_{\text{stat}}\): statistical stability (resampling consistency / significance robustness),
- \(S_{\text{sys}}\): systematic control quality (calibration drift, nuisance sensitivity),
- \(S_{\text{epi}}\): model-assumption robustness (prior/pipeline perturbation stability),
- \(S_{\text{rep}}\): replication concordance (independent analysis agreement).

### RSI Decision Bands
- **RSI ≥ 80**: stable — eligible for progression and broader interpretation.
- **60 ≤ RSI < 80**: conditional — proceed with targeted robustness upgrades.
- **RSI < 60**: unstable — no strong claims; recovery/replication required.

## 4.2 Discovery vs Exclusion KPIs
Track separately to reward null-result progress.

### Discovery KPIs
- confirmed new-signal candidates surviving replication,
- posterior evidence improvement for specific model classes.

### Exclusion KPIs
- number of model classes/operator ranges ruled out per year,
- excluded parameter-volume fraction (with confidence level).

## 4.3 Hypothesis-Test Acceleration KPI
\[
\Delta T_{\text{test}} = t_{\text{first decisive empirical constraint}} - t_{\text{prediction publication}}
\]
Lower is better.

## 5) Ranked Roadmap (Rapid Iteration)

1. **Phase 1: Precision + Null Tests**  
   Focus on sub-3-year cycle experiments; maximize fast constraint generation.

2. **Phase 2: Reproducibility + Modular Upgrades**  
   Standardize data interfaces/toolchains; prioritize upgradeable instruments over monolithic replacements.

3. **Phase 3: EFT Discipline Integration**  
   Enforce RG-consistent constraints and observable-linked model reporting across theory programs.

4. **Phase 4: Cross-Field Talent Ramp**  
   Problem-centered teams (dark sector methods, nonequilibrium EFT, inverse cosmological inference), with adjacent-skill training.

## 6) Kill Criteria Table (Retire vs Refine)

| Condition | Action |
|---|---|
| Null-test excludes core model prediction at predefined confidence | **Retire model class variant** |
| RSI < 60 for two consecutive cycles after remediation | **Pause claims; redesign pipeline** |
| Signal vanishes under independent blinded replication | **Retire discovery claim** |
| Observable proxy depends on unbounded fine-tuning | **Demote priority; require reformulation** |
| Model lacks explicit validity envelope / cutoff transparency | **Reject from funded priority track** |
| Conflicting channels persist beyond uncertainty budget reconciliation | **Split hypotheses; constrain each separately** |

## 7) 12-Month Execution Plan with Decision Gates

## Q1 — Definition & Publication
- select 3 falsifiable GR↔QM interface questions,
- publish observable proxies, null tests, cutoff assumptions, exclusion thresholds.

**Gate G1 (end Q1):** all 3 questions pass EFT-reporting completeness checklist.

## Q2 — Rapid-Cycle Launch
- launch one precision-frontier modular effort,
- publish open benchmark dataset + baseline pipeline.

**Gate G2 (end Q2):** first constraint report + preliminary RSI generated.

## Q3 — Independent Replication
- conduct secondary blinded/semi-blinded replication,
- publish discrepancy matrix + updated uncertainty budget.

**Gate G3 (end Q3):** replication concordance measured; RSI updated with \(S_{rep}\).

## Q4 — Evaluation & Pivot/Proceed
- compute annual Discovery/Exclusion KPIs,
- evaluate \(\Delta T_{test}\), parameter volume excluded, and RSI trend.

**Gate G4 (end Q4):** proceed / pivot / retire decisions issued per kill criteria.
